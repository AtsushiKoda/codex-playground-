# LLM推論 × コンピュータアーキテクチャ可視化ツール

ブラウザで動く軽量なインタラクティブ可視化です。CPUキャッシュ階層（L1/L2/L3）とSRAM/DRAM/SSDの観点で、LLM推論（1 token decode）のボトルネックを近似的に確認できます。

## できること

- モデルサイズ・コンテキスト長・バッチ・量子化を調整
- L1/L2/L3容量、DRAM/SSD帯域、SSDオフロード比率を調整
- ハードウェアプロファイル（エントリCPU / サーバーCPU / GPU推論ノード）を選択して主要設定を一括反映
- 以下を即時反映
  - 推定 TTFT
  - 推定 decode latency / token
  - 推定 tokens/sec
  - ボトルネック判定（Cache/DRAM/SSD）
- メモリ階層ごとの bytes/token フロー表示
- L1-L3/SRAM/DRAM統計（hit/miss/待ち時間寄与）
- Compute→L1/L2/L3→DRAM→SSD のアーキテクチャ図パネル（DRAM/SSDアクセス強度を線色・太さで反映）

## 起動

静的ファイルのみです。次のどちらかで開けます。

```bash
python3 -m http.server 8000
# http://localhost:8000 を開く
```

または `index.html` を直接ブラウザで開いても動作します。

## 注意

### ハードウェアプロファイルの想定用途

- **エントリCPU**: ノートPC・小規模サーバー相当の控えめな帯域/キャッシュを想定。学習用に「DRAM/SSDがボトルネック化しやすい構成」を試す用途。
- **サーバーCPU**: 一般的なサーバー推論ノードを想定。バランス型の比較ベースラインとして利用。
- **GPU推論ノード**: 高帯域メモリを持つ推論専用ノードをイメージした設定。オフロードを抑えたケースの観察に向く。

### 教育向け簡略化について

このツールは教育向けの**簡略化モデル**です。実際のCPU/GPUマイクロアーキテクチャ、NUMA、カーネル実装、ランタイム最適化、I/Oスタックなどは大幅に抽象化しています。実機プロファイラ（perf、VTune、Nsight等）で得られる実測値とは一致しません。
